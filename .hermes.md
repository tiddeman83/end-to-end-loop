# Hermes Project Context

This repository develops the public-facing `end-to-end-loop` Agent Skill. The
production skill entry is `SKILL.md`; supporting product references live in
`references/`.

## Hermes Operating Rules

- Before coding, release, or documentation changes, check this repo against
  `origin`/upstream when available and record if freshness is blocked.
- Load `SKILL.md` before making coding, release, or documentation changes.
- Use `AGENTS.md` as the general project bridge.
- Verify installed CAVEMAN companion skills are present/current before
  code-producing work; install/update only in the active Hermes profile unless
  explicitly directed otherwise.
- Prefer lean/standard/deep routing: deterministic scripts or cheap/fast models
  for mechanical work; high-reasoning models for security/deploy/architecture.
- Keep this repository focused on the delivery-loop tool/skill, not private
  operations or office automation.
- Tool-specific adapter behavior belongs in `references/adapters.md`.
- Do not perform live deploys unless the user explicitly approves the target,
  environment, rollback path, and deploy-readiness gates for that task.

## Self-Learning Memory

- Before target-repo work, inspect `.end-to-end-loop/memory.md` and
  `.end-to-end-loop/memory.local.md` if present.
- After material repo work, write compact result logs and safe learning updates
  when file writes are in scope.
- Keep memory CAVEMAN ULTRA compact: `FACT`, `CMD`, `BLOCK`, `PREF`, `RISK`,
  `FIX`, `AVOID`, `NEXT`.
- Never store secrets, tokens, private chat text, raw env values, or bulky logs.
- If uncertain whether a learning is safe to commit, write it to local-only
  memory or ask the user.
